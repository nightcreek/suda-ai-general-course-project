本项目是纯前端课程演示原型，不包含自建后端服务器。API Key 由用户自行输入，并使用 sessionStorage 在当前浏览器会话中临时共享；关闭标签页后清空。模型名称由用户自行填写，不固定为某一个模型。

机器学习模型嵌入方式

为什么不能直接嵌入 .pkcls

Orange 导出的 .pkcls 属于 Python/Orange 生态的序列化模型对象，浏览器不能直接执行。要在 HTML 中运行，需要提取参数或重新用 JavaScript 写成推理函数。

当前做法

逻辑回归模型被转换成 JavaScript 公式：线性组合 + sigmoid。综合成绩预测使用可解释的多特征估计模型，便于课堂展示和结果解释。

AI 分析模块

学习计划与复盘

学习计划、薄弱点分析和练习生成由用户选择的大语言模型实时生成，不再在页面中写死固定内容。

文件问答

浏览器本地读取文本文件，并将文件内容、学生画像和用户问题一起发送给大模型。

数形结合讲题

大模型生成数学讲解和 GeoGebra 命令，页面解析命令后联动画布绘图。

输入特征

学习时长

每日学习投入。

出勤率

课程参与情况。

睡眠时长

作息状态，不是越高越好。

前次成绩

历史表现基线。

纯前端演示限制

1. API Key 不写死在 HTML 中，使用 sessionStorage 临时保存，关闭标签页后清空。\n2. 模型名称由用户自行填写，适合不同用户使用自己 API Key 支持的模型。\n3. 文件由浏览器本地读取，再作为上下文发给用户指定的大模型 API。\n4. 文本文件最稳定，PDF / Word 建议先转文本。\n5. GeoGebra 依赖外部 CDN，现场演示前应提前测试网络。\n6. 正式系统应使用后端保护 API Key 和用户文件。
