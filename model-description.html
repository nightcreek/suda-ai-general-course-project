项目定位

苏大人工智能通识选修课作业。

DOC
本项目是纯前端课程演示原型，不包含自建后端服务器。API Key、AI 平台、接口地址和模型名称由用户自行输入或选择，并使用 sessionStorage 在当前浏览器会话中临时共享；关闭标签页后清空。首页预设常用国内平台接口，其他平台可使用自定义 OpenAI-compatible 接入。

机器学习模型嵌入方式

为什么不能直接嵌入 .pkcls

Orange 导出的 .pkcls 属于 Python/Orange 生态的序列化模型对象，浏览器不能直接执行。要在 HTML 中运行，需要提取参数或重新用 JavaScript 写成推理函数。

当前做法

逻辑回归模型被转换成 JavaScript 公式：线性组合 + sigmoid。综合成绩预测使用可解释的多特征估计模型，便于课堂展示和结果解释。

AI 分析模块

学习计划与复盘

学习计划、薄弱点分析和练习生成由用户选择的大语言模型实时生成，不再在页面中写死固定内容。

文件问答

浏览器本地读取文本文件，并将文件内容、学生画像和用户问题一起发送给大模型。

数形结合讲题

大模型生成数学讲解和 GeoGebra 命令，页面解析命令后联动画布绘图。

AI 平台接入方式

国内常用平台预设

首页预设 DeepSeek、通义千问、豆包、Kimi、智谱、SiliconFlow、MiniMax 等 OpenAI-compatible 接口，用户只需选择平台、填写 Key 和模型名。

OpenRouter / OpenAI

保留 OpenRouter 与 OpenAI 入口，便于不同用户使用自己的模型供应商。

自定义接口

若平台支持 OpenAI Chat Completions 格式，可填写完整 /chat/completions 地址、API Key 和模型名称。

纯前端演示限制

1. API Key 不写死在 HTML 中，使用 sessionStorage 临时保存，关闭标签页后清空。\n2. 平台、接口地址和模型名称由用户自行设置；常用国内平台已预设，其他平台可使用自定义 OpenAI-compatible。\n3. 文件由浏览器本地读取，再作为上下文发给用户指定的大模型 API。\n4. 文本文件最稳定，PDF / Word 建议先转文本。\n5. GeoGebra 依赖外部 CDN，现场演示前应提前测试网络。\n6. 正式系统应使用后端保护 API Key 和用户文件。

Prompt 与 GeoGebra 命令参考

说明 AI 讲题智能体如何生成可执行命令。

项目为不同模块设置了不同 prompt。AI 文件问答会读取学生画像、预测结果和上传文件；学习计划页面会读取画像和预测结果生成计划、薄弱点与练习；AI 数形结合页面会自动加载 assets/docs/geogebra-commands.md，把简化版 GeoGebra 命令参考拼入 system prompt，要求模型把可执行命令放在 [GGB_COMMANDS_START] 与 [GGB_COMMANDS_END] 标签中。前端解析命令后调用 GeoGebra evalCommand() 自动绘图。
